Subscribing to a SRFI List

There are individual mailing lists that are only active for the period when each SRFI is in draft state. All discussion of a SRFI proposal should take place here, as not all interested parties participate in other venues such as comp.lang.scheme.

To subscribe to the mailing list for SRFI number n, send mail to <srfi-n-request@srfi.schemers.org> with the subject reading subscribe.

To unsubscribe, send mail to the same address with the subject reading unsubscribe.

In addition to the lists mentioned above, there is a moderated mailing list <srfi-announce@srfi.schemers.org> where all announcements about new SRFI proposals and changes in status are posted. To subscribe to this list, send mail to <srfi-announce-request@srfi.schemers.org> with the subject reading subscribe.

To unsubscribe, send mail to the same address with the subject reading unsubscribe.

Moreover, there is a pseudo-mailing list <srfi-auto-subscribe@srfi.schemers.org>. Its subscribers automatically become subscribers of all future discussion mailing lists for Scheme Requests for Implementation. Therefore, there are no messages on this list itself. To subscribe, send mail to <srfi-announce-request@srfi.schemers.org> with the subject reading subscribe.

To unsubscribe, send mail to the same address with the subject reading unsubscribe.
